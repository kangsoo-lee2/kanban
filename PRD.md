## 1. 제품 개요

### 1.1 제품명
**칸반 보드 (Kanban Board)**

### 1.2 한 줄 설명
드래그앤드롭으로 작업을 세 단계(To-Do / In-Progress / Done)에 걸쳐 시각적으로 관리하는 웹 앱. Google·GitHub·이메일 계정으로 로그인하여 어디서든 데이터를 동기화.

### 1.3 배경 및 목적
- v1.0은 localStorage 단일 사용자 앱으로 다기기 동기화가 불가능했음.
- v2.0은 Supabase 인증과 PostgreSQL 기반 저장소를 도입해 사용자별 데이터를 영속적으로 관리.
- GitHub Pages에 배포하여 별도 서버 없이 정적 앱으로 운영.

### 1.4 배포 주소
**https://kangsoo-lee2.github.io/kanban/**

### 1.5 목표
| # | 목표 | 측정 기준 |
|---|------|-----------|
| G1 | 카드 이동 조작 ≤ 3초 (네트워크 포함) | 드래그 완료 ~ 재렌더 시간 |
| G2 | 페이지 로드 ≤ 2초 | Lighthouse Performance ≥ 85 |
| G3 | 데이터 영속성 100% | 다기기 접속 후 동일 카드 목록 확인 |
| G4 | 모바일·데스크톱 모두 정상 표시 | 320px ~ 1920px 너비 |
| G5 | 인증 3종 모두 동작 | Google / GitHub / 이메일 로그인 정상 확인 |

## 2. 사용자 정의

### 2.1 주요 사용자
| 페르소나 | 설명 | 핵심 니즈 |
|----------|------|-----------|
| 개인 개발자 | 여러 기기에서 프로젝트를 관리하는 개발자 | 소셜 로그인, 데이터 동기화 |
| 수강생 | 교육 과정에서 실습 과제를 관리 | 이메일 회원가입, 단순한 UI |
| 소규모 팀원 | 팀 작업 현황을 공유하며 협업 | 계정별 독립 보드 |

### 2.2 사용자 시나리오

#### 신규 사용자
1. 배포 URL 접속 → 로그인 화면
2. Google/GitHub OAuth 또는 이메일 회원가입 선택
3. 인증 완료 → 칸반 보드 접속 (빈 보드)
4. 카드 추가 → Supabase DB 저장

#### 기존 사용자
1. 배포 URL 접속 → 세션 자동 복원 → 칸반 보드 바로 표시
2. 이전에 저장한 카드 자동 로드
3. 카드 드래그앤드롭 → 변경 사항 즉시 DB 반영

## 3. 기능 요구사항

### 3.1 필수 기능 (Must Have)

#### F-01 칼럼 표시
- `To-Do`, `In-Progress`, `Done` 세 칼럼을 좌→우 순서로 표시한다.
- 각 칼럼 헤더에 현재 카드 수를 배지(badge)로 표시한다.

#### F-02 카드 추가
- 각 칼럼 하단에 텍스트 입력 영역과 `+ 추가` 버튼을 제공한다.
- `Enter` 키(Shift+Enter 제외)로도 추가 가능하다.
- 빈 문자열은 추가하지 않는다.
- 추가된 카드는 Supabase `cards` 테이블에 저장된다.

#### F-03 카드 삭제
- 카드 우측 `✕` 버튼 클릭 시 해당 카드를 삭제한다.
- Supabase DB에서 해당 행을 즉시 삭제한다.

#### F-04 드래그앤드롭 이동
- HTML5 Drag and Drop API를 사용해 카드를 다른 칼럼으로 이동한다.
- 이동 시 Supabase DB의 `col` 필드가 업데이트된다.

#### F-05 인증 — Google OAuth
- Supabase `signInWithOAuth({ provider: 'google' })`를 사용한다.
- 인증 후 칸반 보드로 리다이렉트된다.

#### F-06 인증 — GitHub OAuth
- Supabase `signInWithOAuth({ provider: 'github' })`를 사용한다.
- 인증 후 칸반 보드로 리다이렉트된다.

#### F-07 인증 — 이메일/비밀번호
- 신규 사용자: `signUp` → 확인 이메일 발송.
- 기존 사용자: `signInWithPassword` → 즉시 로그인.

#### F-08 세션 관리
- `onAuthStateChange`로 세션 상태를 감지한다.
- 세션이 있으면 보드 화면, 없으면 로그인 화면을 표시한다.
- 로그아웃 버튼으로 세션을 종료한다.
